# Type Check Skill

型チェック実行時は **必ずエージェント（Task tool）を使用** する。直接Bashで `cargo check` を実行しない（長い出力で親のコンテキストを消費しないため）。

## 実行方法

Task tool を使用:
- subagent_type: `type-check-agent`（本プラグインの agents/type-check-agent.md がカスタムエージェントとして登録される）
- prompt にユーザーの依頼内容を具体的に含める:

```
Task tool:
  subagent_type: "type-check-agent"
  prompt: |
    以下のRustプロジェクトの型チェックを実行して結果を報告してください。

    プロジェクト: {プロジェクトのパス。不明ならカレントディレクトリ}
    対象: {全体 / 特定のパッケージ}
    補足: {警告もエラー扱いで厳密にチェック、などの要望があれば}
```

サブエージェントは親の会話コンテキストを引き継がないため、対象や要望は必ずpromptに明示する。

## 期待される結果

エージェントから以下の報告を受け取り、ユーザーに要約して伝える：

- 成功/失敗サマリー（エラー件数・警告件数。警告のみの場合も内容を含む）
- エラーの詳細（ファイルパス、行番号、メッセージ、rustcのhelp、修正提案）

エラーがあった場合は、報告された内容をもとに修正方針を立ててから次のアクションに進む。
