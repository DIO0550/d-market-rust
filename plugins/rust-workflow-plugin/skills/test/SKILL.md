# Test Skill

テスト実行時は **必ずエージェント（Task tool）を使用** する。直接Bashで `cargo test` を実行しない（長い出力で親のコンテキストを消費しないため）。

## 実行方法

Task tool を使用:
- subagent_type: `test-agent`（本プラグインの agents/test-agent.md がカスタムエージェントとして登録される）
- prompt にユーザーの依頼内容を具体的に含める:

```
Task tool:
  subagent_type: "test-agent"
  prompt: |
    以下のRustプロジェクトのテストを実行して結果を報告してください。

    プロジェクト: {プロジェクトのパス。不明ならカレントディレクトリ}
    対象: {全テスト / 特定のテスト名・モジュール・パッケージ}
    補足: {--nocapture が必要、ドキュメントテストも実行、などの要望があれば}
```

サブエージェントは親の会話コンテキストを引き継がないため、対象や要望は必ずpromptに明示する。

## 期待される結果

エージェントから以下の報告を受け取り、ユーザーに要約して伝える：

- 成功/失敗サマリー（passed / failed / ignored 件数、テスト0件ならその旨）
- 失敗テストの詳細（テスト名、パニックメッセージ、修正提案）
- コンパイルエラーの場合はビルド失敗としての詳細

失敗があった場合は、報告された内容をもとに修正方針を立ててから次のアクションに進む。
